readfile

Formatted documentation for the readfile function.

Read a UTF-8 or ANSI (US-ASCII) file.

This function is aimed at providing a reliable method of reading a file. The backbone of this function is the fileread function. Further processing is done to attempt to detect if the file is UTF-8 or not, apply the transcoding and returning the file as an n-by-1 cell array for files with n lines.

The test for being UTF-8 can fail. For files with chars in the 128:255 range, the test will often determine the encoding correctly, but it might fail. Online files are much more limited than offline files. To avoid this the files are downloaded to tempdir() and deleted after reading.

In Octave there is poor to no support for chars above 255. This has to do with the way Octave runs: it stores chars in a single byte. This limits what Octave can do regardless of OS. There are plans to extent the support, but this appears to be very far down the priority list, since it requires a lot of explicit rewriting. Even the current support for 128-255 chars seems to be . (Note that this paragraph was true in August 2020, so a big update to Octave (e.g. Octave 6) may have added support by now. Although, don't hold your breath.)

Syntax

data=readfile(filename)
data=readfile(url)

Output arguments

data An n-by-1 cell array. it contains 1 cell per line in the file, even for empty lines.
See the compatibility matrix below for details about which combinations of OS, runtime and version are expected to return the correct output.

Input arguments

filename The relative or absolute path to a file.
url The url to a file.
An attempt will be made to download the file to the folder returned by tempdir(). After the reading is completed (or fails) the file is deleted.
